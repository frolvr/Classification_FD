# Classification_FD

Классификация обращений в службу поддержки с использованием TF-IDF и логистической регрессии.

## Описание

Проект предназначен для автоматической классификации текстовых обращений (тикетов) службы поддержки. На вход подаются описания обращений, а модель предсказывает категорию (тег) на основе обученной базы знаний.

Классификация основана на методе **TF-IDF** (Term Frequency — Inverse Document Frequency) для векторизации текста и **логистической регрессии** (Logistic Regression) для классификации.

## Методология

### Предобработка текста

- Приведение к нижнему регистру
- Разбиение текста на предложения по знакам препинания (`.`, `?`, `!`)
- Удаление стоп-слов (русский язык, через NLTK)
- Лемматизация (через pymorphy2)
- Удаление чисел и знаков пунктуации

### Обучение модели

1. **LabelEncoder** — преобразование текстовых категорий в числовые метки.
2. **TfidfVectorizer** — преобразование текста в числовые векторы TF-IDF с параметрами:
   - `ngram_range=(1, 2)` — униграммы и биграммы
   - `max_features=20000` — максимальное количество признаков
   - `min_df=2`, `max_df=0.75` — фильтрация по частоте
3. **LogisticRegression** — мультиклассовая классификация (solver=`lbfgs`, multi_class=`multinomial`).
4. **Pipeline** — объединение всех шагов в единый пайплайн.

### Оценка качества

Каждый предсказанный результат сопровождается оценкой уверенности (`Score`). Применяется пороговая фильтрация:

| Файл | Описание |
|------|----------|
| `clean.xlsx` | Тикеты с `Score ≥ 0.3` — классифицированы уверенно |
| `basket.xlsx` | Тикеты с `Score < 0.3` — требуют ручной проверки и переклассификации |

### Требования

- pandas
- numpy
- scikit-learn

### Входные данные

- **Обучающая выборка** (`БК-База-знаний111.xlsx`):
  - Колонка `Вопрос` — текст вопроса/обращения
  - Колонка `Тип-подкатегория-подподкатегория` — иерархическая категория

- **Тестовая выборка** (`tickets-August_after_processing.xlsx`):
  - Колонка `opisanie` — описание тикета
  - Остальные колонки сохраняются в выходных файлах

### Выходные данные

Два Excel-файла:
- `clean.xlsx` — успешно классифицированные обращения с колонками `Теги` (предсказанная категория) и `Score` (уверенность)
- `basket.xlsx` — обращения, требующие ручной обработки
